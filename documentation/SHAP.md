# SHAP Explanations

SHAP (SHapley Additive exPlanations) is used in the IntelliCloud dashboard to provide a human-readable visual explanation of the AI's decision process for each task classification.

## 1. What is SHAP?
SHAP is a game-theoretic approach to explain the output of any machine learning model. In IntelliCloud, it explains why the Random Forest model chose one of the 5 Energy Efficiency Classes.

## 2. How the Image is Formed
The SHAP explanation image (e.g., `api/shap_explanations/explanation_...png`) is generated by `core/start.py`'s `generate_shap_explanation` method:

1. **Calculating SHAP Values**: The `shap.TreeExplainer` is used to calculate the contribution of each of the 12 features for every possible class (1 through 5).
2. **Identifying Top Classes**: The system identifies the **Chosen Class** (highest probability) and the **Runner-up Class** (second highest probability).
3. **Feature Selection**: It selects the 10 most influential features that contributed to the classification of these two classes.
4. **Plotting the Bars**:
    - **Teal Bars**: Represent the SHAP values for the **Chosen Class**.
    - **Orange Bars**: Represent the SHAP values for the **Runner-up Class** (at 50% transparency for contrast).

## 3. How SHAP Works to Form the Image
The image is a **horizontal bar chart** where:
- The **length of the bar** indicates the magnitude of the impact.
- **Positive SHAP Value (Right of Zero)**: The feature's value **increased** the probability of the AI choosing that class.
- **Negative SHAP Value (Left of Zero)**: The feature's value **decreased** the probability of the AI choosing that class.

### 4. Key Components of the SHAP Image
- **Task Header**: Shows the unique identifier for the specific task processed.
- **Main Plot**: Shows the comparison between the chosen class and the runner-up class.
- **Sidebar (Guide for Beginners)**: A human-readable text box that explains:
    - **How to read the report**.
    - **What the colors represent**.
    - **What the biggest factor was** for that specific classification.

### 5. Why SHAP is Used
- **Transparency**: It allows users (and developers) to see **exactly why** a task was classified as "Low Efficiency" (e.g., "High Power Consumption" and "Short Execution Time").
- **Trust**: Seeing the underlying data-driven justification for the AI's choice builds trust in the IntelliCloud orchestration layer.
- **Debugging**: If a task is misclassified, the SHAP plot can help identify if the feature extraction or model training is biased.

### 6. Example Features explained by SHAP
- **CPU Focus**: If this bar is far to the right for Class 1 (Low Efficiency), it means high CPU usage was the main reason the task was classified as inefficient.
- **Input Size**: If this bar is far to right for Class 5 (High Efficiency), it means the task was handled very efficiently despite its large input size.
